# Generated adapter contract

A generated adapter package includes:

- `target.yaml` with versioned provider-neutral metadata;
- `authorization.example.yaml` with no claim of verified permission;
- `capabilities.yaml` with all unimplemented capabilities disabled;
- an adapter class implementing only the provider-neutral protocol stubs;
- an acceptance assertion stub owned by the first-party integration;
- contract and negative test skeletons;
- a README and scaffold manifest with generated file hashes.

All generated methods fail closed until implemented and validated. The scaffold manifest uses `generated_not_executed`.
