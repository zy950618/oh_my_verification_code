# Output risk profile

## High-risk output failures

- Calling a model prediction, provider verification, browser message, token, or HTTP 200 “business success.”
- Omitting authorization state or silently upgrading an unverified claim.
- Returning executable actions without a fresh challenge binding, coordinate frame, expiry, and stop conditions.
- Generating partner-specific files into the public package.
- Treating driver completion as a first-party backend receipt.
- Exposing raw captures, cookies, tokens, profiles, or private target identifiers.

## Controls

- Canonical typed status fields and a single promotion decision function.
- Fact labels and missing-evidence lists in every high-risk result.
- Generate-only target scaffolding with zero network and driver calls.
- Optional adapters that cannot construct business receipts.
- Route, output, success-semantics, private-artifact, and evidence-migration tests.
